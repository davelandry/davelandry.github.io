Design Principles

Design Principles

There are 4 primary attributes to encode data visually:

  • Size
  • Color
  • Shape
  • Position

Size

  • Circle radius
  • Rectangle width/height
  • Arc radius
  • Line/stroke width

Color

  • Hue
  • Saturation
  • Lightness
  • Opacity

Shape

  • Geometry
  • Iconography

Position

  • X/Y
  • Rotation

Visual Narrative

Guided vs Exploratory

Guided Narrative

Visualizations are used to tell
a directed story or to make a point.

Think one page scrollers or visualizations
embedded in news articles.

Defining Characteristic

Tend to generally have a stronger focus on design and mood, as there is no cumbersome utilitarian interface to get in the way.

Exploratory Narrative

Visualizations that foster a free user-experience, letting them draw their own hypotheses and conclusions.

Usually some sort of sandbox or interface
with a lot of user defined control.

Defining Characteristic

Stronger attention needs to be paid to the organization of user interface elements, to make them both visually unobstrusive and easy to learn.

Scales

“Scales are a convenient abstraction for a fundamental task in visualization: mapping a dimension of abstract data to a visual representation. Although most often used for position-encoding quantitative data, such as mapping a measurement in meters to a position in pixels for dots in a scatterplot, scales can represent virtually any visual encoding, such as diverging colors, stroke widths, or symbol size.”

Linear Scales

As an example, let's say you have
a dataset containing travel distances: 

// distances are in miles
var data = [0.2, 2.4, 4.7, 3.1];

var xScale = d3.scaleLinear()
  .domain([0, 5])
  .range([0, window.innerWidth]);

var firstX = xScale(data[0]);
