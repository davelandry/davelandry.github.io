Design Principles

Design Principles

There are 4 primary attributes to encode data visually:

  • Size
  • Color
  • Shape
  • Position

Size

  • Circle radius
  • Rectangle width/height
  • Arc radius
  • Line/stroke width

Color

  • Hue
  • Saturation
  • Lightness
  • Opacity

Shape

  • Geometry
  • Iconography

Position

  • X/Y
  • Rotation

Visual Narrative

Guided vs Exploratory

Guided Narrative

Visualizations are used to tell
a directed story or to make a point.

Think one page scrollers or visualizations
embedded in news articles.

Defining Characteristic

Tend to generally have a stronger focus on design and mood, as there is no cumbersome utilitarian interface to get in the way.

Some Good Examples

The Pudding

Exploratory Narrative

Visualizations that foster a free user-experience, letting them draw their own hypotheses and conclusions.

Usually some sort of sandbox or interface
with a lot of user defined control.

Defining Characteristic

Stronger attention needs to be paid to the organization of user interface elements, to make them both visually unobstrusive and easy to learn.

FiveThirtyEight

Homework

Whether storing in a file on your computer or using an API, set up an HTML page that loads the data you need for your final project into Javascript using D3. Use console.log to display your data and we will look at it next class all together.

Don't worry about drawing anything based on the data, this exercise is solely for exploring the data in the console.

Homework Example

<!DOCTYPE html>
<html>
  <head>
    <meta charset="utf-8" />
    <script src="https://d3js.org/d3.v7.min.js"></script>
  </head>
  <body>
    <script>
      d3.csv("pokemonStats.csv")
        .then(function(data) {
          console.log(data);
        });
    </script>
  </body>
</html>
