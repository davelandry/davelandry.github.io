Geography Data Format

GeoJSON is an open standard format designed for representing simple geographical features. It is based on JSON, the JavaScript Object Notation.

Let's take a look at the examples in Wikipedia:
https://en.wikipedia.org/wiki/GeoJSON

Loading Geographies

TopoJSON files are loaded the same way as any other JSON data file:

d3.json("world-alpha3.json", function(error, world) {
  console.log(world);

1 More Library

Put this code in between your head tags:

<script src="https://unpkg.com/topojson@3"></script>

TopoJSON to GeoJSON

The loaded TopoJSON Object must be converted to GeoJSON, after loading, before we are able to use it for drawing:

var geoJSON = topojson.feature(world, world.objects.countries);

Map Projections

Similar to setting up a linearScale for mapping data values to pixel positions, we need to create a helper function that maps raw coordinate values into to screen pixels.

This is why projections matter.

Set up a Projection

Choose one of the projections included with D3 from here.

var proj = d3.geoMercator()
  .fitSize([width, height], geoJSON);

The mercator projection is the most commonly used projection for displaying the world.

Path Constructor

Since geographies are often very complex shapes, we can use the "geoPath" function to create a helper function that will use our projection and translate the pixel values into a valid "d" attribute for each shape:

var path = d3.geoPath()
  .projection(proj);

Draw the Geographies

And finally, bind the geoJSON features to an enter event! The "path" variable can be passed a GeoJSON feature, and it will return an SVG path.

var countries = map.selectAll("path")
  .data(geoJSON.features);

countries.enter().append("path")
  .attr("d", path);

Filter the Geographies

Sometimes you don't want to display every geography in a TopoJSON file. In world maps, it's common to hide Antarctica (sorry Antarctica). Simply apply a filter to the geoJSON features array:

geoJSON.features = geoJSON.features.filter(function(d) {
  return d.id !== "ATA";
});

Color the Geographies

Geographies (or rather, "paths") are styled the exact same way as any other SVG shape:

countries.enter().append("path")
  .attr("d", path)
  .attr("fill", "green");

What about the ocean?

Easy, just put a rectangle in the background! Add this into your SVG, above the map group:

<rect id="ocean" fill="cornflowerblue" />

And then use JavaScript to set the correct size:

svg.select("#ocean")
  .attr("width", width)
  .attr("height", height);

Coordinate Points?

The "path" variable can also be passed a coordinate point, and it will return it's pixel position. Be careful, it expects lat/long in the opposite order!

var points = [
  {"name": "Boston", "coords": [-71.0589, 42.3601]},
  {"name": "London", "coords": [-0.1278, 51.5074]}
];

var points = map.selectAll("circle")
  .data(points);

points.enter().append("circle")
  .attr("transform", function(d){
    return "translate(" + proj(d.coords) + ")";
  })
  .attr("r", 10);

What about Zooming?

Similar to d3 axes, we create a "zoom function" and call it on the entire SVG. This function will listen for user scroll and panning, and adjust the map accordingly.

var zoom = d3.zoom()
  .translateExtent([[0, 0], [width, height]])
  .scaleExtent([1, 8])
  .on("zoom", zoomed);

function zoomed() {
  map.attr("transform", d3.event.transform);
}

svg.call(zoom);
